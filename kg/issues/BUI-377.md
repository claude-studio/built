## 목표

`.built/config.json`의 `default_run_profile.providers`가 새 run-request 생성 경로와
`/built:run` 실행 fallback에 일관되게 반영되도록 한다.

## 구현 내용

- `src/providers/presets.js`의 run-request 생성 helper가 provider preset, 직접 providers,
  `defaultRunProfile`을 모두 ProviderSpec snapshot으로 정규화하도록 보강했다.
- `src/state.js`의 `initRunRequest`가 대상 프로젝트 `.built/config.json.default_run_profile`을
  읽어 `/built:plan` run-request 생성 경로에 반영하도록 연결했다.
- `scripts/run.js`는 run-request에 `providers` 필드가 없을 때 config default profile을
  fallback으로 적용하고, provider 선택 근거를 stdout, `state.json`, `root-context.json`에
  기록한다.
- `skills/plan/SKILL.md`에 `initRunRequest`가 default profile을 snapshot으로 정규화한다는
  설명을 추가했다.

## 결정 내용

- config의 `default_run_profile.providers`는 BUI-317 계약대로 provider name 문자열 맵을
  유지한다.
- run-request와 runtime state/root-context에는 실행 시점의 normalized ProviderSpec snapshot을
  남긴다.
- 명시적인 `run-request.providers`가 없을 때만 config default profile을 fallback으로 적용한다.
  잘못된 providers 타입이나 phase는 fallback으로 숨기지 않고 provider 설정 오류로 중단한다.
