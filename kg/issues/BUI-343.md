## 목표

`scripts/plan-draft.js`가 plugin repo root를 project root로 고정해 Plan/Design draft를 잘못된 `.built/runs` 아래에 저장할 수 있는 문제를 제거한다.
완료 기준은 plugin 외부 target cwd에서 draft를 만들 때 target project `.built/runs`에 저장되고, 기존 repo-local 개발 실행이 회귀하지 않는 것이다.

## 구현 내용

- `scripts/plan-draft.js`의 root 해석을 `options.projectRoot`/문자열 옵션, argv `--project-root`, `BUILT_PROJECT_ROOT`, `process.cwd()` 순서로 바꿨다.
- plugin script를 절대 경로로 require하는 환경에서도 기본값은 실행자의 target project cwd가 되도록 `__dirname` 기반 plugin repo root 고정을 제거했다.
- `test/plan-draft.test.js`에 target project fixture 기반 저장 위치 검증과 argv `--project-root`가 cwd보다 우선하는 회귀 테스트를 추가했다.
- `docs/contracts/file-contracts.md`와 `skills/plan/SKILL.md`에 target project root 계약, `BUILT_PROJECT_ROOT`, `{ projectRoot }`, `node -e ... -- --project-root <path>` 사용법을 문서화했다.

## 결정 내용

- plan draft 저장 위치는 plugin repo가 아니라 target project root 기준으로 고정한다.
- 기본 target project root는 `process.cwd()`로 둔다.
  `/built:plan`은 사용자가 작업 중인 프로젝트의 산출물을 다루는 skill이며, plugin cache 또는 plugin repo의 runtime 파일을 소유하지 않기 때문이다.
- cwd를 보장할 수 없는 runner는 `BUILT_PROJECT_ROOT`, `{ projectRoot }`, 또는 `node -e ... -- --project-root <path>`를 명시해야 한다.
- `node -e` argv 경로는 Node 옵션과 script argv를 구분하기 위해 `--` separator를 필수 운영 계약으로 남긴다.

## 결정 이유

- `__dirname`은 helper 파일의 위치를 가리키므로 marketplace/plugin cache 설치 환경에서 target project와 분리된다.
- Plan/Design draft는 세션 복구용 사용자 프로젝트 산출물이므로 target project `.built/runs/<feature>/plan-draft.md`에 있어야 다음 `/built:plan` 실행이 같은 상태를 찾을 수 있다.
- cwd 기본값은 기존 repo-local 개발 실행과 skill 사용법을 유지하면서 절대 경로 require 환경도 지원한다.
- 명시 root escape hatch를 여러 형태로 둔 이유는 runner, direct API, shell `node -e` 호출이 각각 root를 전달하는 방식이 다르기 때문이다.
