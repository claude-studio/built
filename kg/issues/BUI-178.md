## 목표

provider 비교 모드 MVP는 fake/offline 중심으로 유지하되, 실제 Claude/Codex 결과 차이를 확인할 수 있는 별도 opt-in smoke entrypoint를 추가한다.
기본 테스트와 CI를 인증, 네트워크, 비용 의존 경로로 오염시키지 않으면서 candidate별 artifact와 비교 report를 남기는 것이 핵심 목표다.

## 구현 내용

- `scripts/smoke-compare-providers.js`를 추가해 `BUILT_COMPARE_REAL_SMOKE=1`일 때만 Claude/Codex real comparison smoke를 실행하게 했다.
- 환경 변수가 없으면 skip 메시지 출력 후 exit 0으로 종료한다.
- smoke는 임시 git repo를 만들고 `compare-smoke` feature spec과 `run-request.json`을 생성한 뒤 `scripts/compare-providers.js`를 `do` phase로 실행한다.
- candidate는 `claude`, `codex` 두 개로 고정하고, Codex candidate는 `workspace-write` sandbox와 15분 provider timeout을 사용한다.
- 전체 smoke 실행은 40분 timeout으로 제한한다.
- 비교 산출물은 `.built/runtime/runs/compare-smoke/comparisons/<comparison-id>/` 아래에서 검증한다.
  `report.md`, `manifest.json`, candidate별 `state.json`, `verification.json`, `diff.patch`, `git-status.txt`, `run-request.json` 존재를 확인한다.
- canonical `.built/features/compare-smoke/do-result.md`가 생성되지 않았는지 확인해 비교 모드가 canonical output을 오염시키지 않는지 검증한다.
- `package.json`에 `npm run test:smoke:compare`를 추가했다.
- `docs/smoke-testing.md`에 실행 전제, opt-in 명령, 산출물 구조, 실패 원인축, cleanup 절차, 비범위를 추가했다.

## 결정 내용

- real comparison smoke는 기본 `npm test`에 포함하지 않는다.
  사용자가 `BUILT_COMPARE_REAL_SMOKE=1` 또는 `npm run test:smoke:compare`로 명시적으로 선택할 때만 실행한다.
- smoke 범위는 Claude/Codex candidate 각각 하나와 `do` phase 하나로 제한한다.
- 자동 winner 선택, 자동 merge, canonical branch 적용은 수행하지 않는다.
  `report.md`와 candidate별 diff/verification artifact만 사람이 판단할 evidence로 남긴다.
- 실제 provider 실패는 candidate별 artifact와 report로 분리해서 남기며, 실패 원인축은 smoke 문서의 taxonomy와 맞춘다.
- `BUILT_KEEP_SMOKE_DIR=1`을 지원해 비교 산출물을 수동 inspection용으로 유지할 수 있게 한다.

## 결정 이유

- real provider 비교는 인증, provider availability, 네트워크, 비용, quota 상태에 의존하므로 기본 회귀 테스트의 신뢰도와 재현성을 낮출 수 있다.
- fake comparison E2E는 file contract와 canonical 보호의 기본 회귀 신호이고, real smoke는 provider별 품질 gap과 실제 CLI 통합 문제를 확인하는 선택적 운영 신호다.
- candidate별 output directory를 검증해야 한 provider의 실패나 diff가 다른 provider 또는 canonical 결과로 섞이는 문제를 조기에 잡을 수 있다.
- 자동 winner는 검증 명령 성공만으로 품질과 유지보수성을 판단하기 어렵기 때문에 아직 운영 정책으로 승인하지 않는다.

## 발생한 이슈와 review history

- Builder는 `npm test` 39개 통과를 보고했고, 기본 테스트가 `test:smoke:compare`를 호출하지 않는다고 확인했다.
- Reviewer Round 1은 2026-04-26 16:00 KST에 PASS로 판정했다.
  신규 격리 smoke script이며 core runner 변경이 없고, opt-in guard, candidate artifact 격리, canonical output 보호, 문서 갱신이 완료 기준을 충족한다고 봤다.
- Reviewer는 비차단 관찰 사항으로 cleanup 동작 불일치를 기록했다.
  Node.js에서 `process.exit()` 호출은 `finally`의 `cleanup()` 실행을 보장하지 않으므로, 문서의 "smoke 종료 시 자동 삭제"와 실제 동작이 어긋날 수 있다.
  opt-in 전용 smoke이고 임시 디렉토리 경로라 즉시 blocker는 아니지만, 후속 이슈에서 `process.on('exit', cleanup)` 또는 명시적 return-based exit 처리로 보정하는 것을 권장했다.

## 재발 방지 포인트

- real comparison smoke를 기본 `npm test` 또는 기본 CI path에 넣지 않는다.
- `BUILT_COMPARE_REAL_SMOKE=1` opt-in guard를 제거하거나 기본값을 real 실행으로 바꾸지 않는다.
- comparison smoke artifact는 candidate별 `providers/<candidate-id>/` 아래에 남겨야 하며, canonical `.built/features/<feature>/`에 결과 파일을 쓰면 회귀로 본다.
- `report.md`에는 자동 winner를 선택하지 않았다는 문구가 있어야 한다.
- cleanup 동작을 고칠 때는 `process.exit()`가 `finally`를 우회하는 점을 고려한다.
  자동 삭제 문서를 유지하려면 exit path를 return 기반으로 정리하거나 `process.on('exit')` cleanup을 별도로 등록한다.
- `BUILT_KEEP_SMOKE_DIR=1`은 investigation용 escape hatch로 유지하되, handoff나 문서에는 수동 삭제 절차를 함께 남긴다.
