## 목표

사용자가 helper script를 직접 실행하지 않고 `/built:run-codex-do <feature>` 명령으로
`codex-do` provider preset을 적용한 뒤 전체 run을 시작할 수 있게 한다.

## 구현 내용

- `skills/run-codex-do/SKILL.md`를 추가해 user-invocable command를 제공했다.
- 스킬은 대상 프로젝트 cwd를 유지하고 `SCRIPT_DIR` 절대 경로로 `provider-preset.js <feature> --preset codex-do`와 `run.js <feature>`를 실행하도록 문서화했다.
- `--background` 실행 안내를 포함해 `run.js <feature> --background` 경로를 지원한다.
- `skills/run/SKILL.md` 변형 표와 README에 Codex opt-in 실행 경로를 추가했다.
- packaging test에 신규 skill 존재, `SCRIPT_DIR` 절대 경로 호출, target cwd 유지, `codex-do` Do/Iter `workspace-write`, background 안내 검증을 추가했다.
- provider preset helper와 plugin packaging workflow KG 문서에 `run-codex-do` 검증 대상을 반영했다.

## 결정 사항

- Codex를 전체 기본 provider로 바꾸지 않고 Do/Iter만 Codex로 opt-in하는 `codex-do` preset 전용 스킬을 먼저 노출한다.
- plugin cache cwd로 이동하지 않고 사용자가 실행한 target project cwd를 유지한다.
- 실제 Codex provider smoke는 이번 변경의 merge blocker로 두지 않는다.
  이번 범위는 user-facing skill, 문서, packaging 검증이며 provider runtime hotfix는 BUI-306에서 선행 완료됐다.
