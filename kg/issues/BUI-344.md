## 목표

built plugin이 target project 밖에 설치된 상태에서도 Plan, Design, Run 및 관련 skill 문서의 helper 호출이 안정적으로 동작하게 한다.
완료 기준은 target project cwd에 `scripts/` 또는 `src/`가 없어도 plugin scripts를 실행할 수 있고, 문서의 `<BUILT_PLUGIN_DIR>` placeholder가 실제 사용 가능한 절차로 대체되는 것이다.

## 구현 내용

- `skills/run/SKILL.md`, `skills/plan/SKILL.md`, `skills/do/SKILL.md`, `skills/check/SKILL.md`, `skills/init/SKILL.md`, `skills/status/SKILL.md`, `skills/doctor/SKILL.md`에서 target project cwd와 built plugin/repo 경로를 분리했다.
- run preset 문서인 `skills/run-opus/SKILL.md`, `skills/run-sonnet/SKILL.md`, `skills/run-codex/SKILL.md`도 같은 `BUILT_PLUGIN_DIR` 기반 호출 방식으로 정렬했다.
- `node scripts/...`, `require('./scripts/...')`, `require('./src/...')`, `<BUILT_PLUGIN_DIR>` placeholder, `BASH_SOURCE[0]` 기반 plugin dir 추정 예시를 지정 범위에서 제거했다.
- `test/plugin-packaging.test.js`에 target project에 `.built`/`.claude`만 있고 `scripts/`, `src/`가 없는 dogfooding fixture를 추가했다.
- fixture는 bash와 zsh에서 `BUILT_PLUGIN_DIR` 기반 Plan/Run helper 호출이 target project `.built`에 산출물을 남기는지 검증한다.

## 결정 내용

- skill 문서의 plugin script 호출 표준은 target project cwd 유지와 `BUILT_PLUGIN_DIR` 기반 절대 경로 호출로 고정한다.
- `BUILT_PLUGIN_DIR`가 설정되지 않았을 때는 target cwd 상대 경로로 추정하지 않고 명확한 shell parameter expansion 오류로 실패하게 둔다.
- helper가 target project 산출물을 만들 때는 `process.cwd()` 또는 명시된 project root를 target project 기준으로 사용하고, plugin package 내부에는 runtime `.built`를 만들지 않는다.

## 결정 이유

- marketplace/plugin cache 설치 환경에서는 skill 문서가 실행되는 target project와 plugin 파일 위치가 분리된다.
- `BASH_SOURCE[0]`는 interactive shell, Claude Bash tool, zsh/bash 혼용 환경에서 skill 파일 위치를 안정적으로 가리키는 계약이 아니다.
- target cwd 상대 `scripts/`/`src/` 호출은 사용자의 프로젝트에 built repository layout이 있다고 가정하므로 dogfooding과 실제 설치 환경에서 깨진다.
- `BUILT_PLUGIN_DIR`는 plugin 파일 위치를 명시적으로 주입하고, target project cwd는 사용자 산출물 위치로 유지해 file contract와 runner/control plane 책임을 침범하지 않는다.

## 재발 방지 포인트

- skill 문서가 plugin helper를 호출할 때 target project cwd의 `scripts/` 또는 `src/` 존재를 가정하지 않는다.
- interactive shell 예시는 `BASH_SOURCE[0]`로 plugin dir를 역추적하지 말고 `BUILT_PLUGIN_DIR`를 필수 입력으로 둔다.
- packaging smoke에는 `.built`/`.claude`만 있는 target project fixture를 유지해 repository-root에서만 통과하는 호출 예시를 잡는다.
- `BUILT_PLUGIN_DIR` unset 실패는 의도된 guard다. 조용히 target cwd 상대 fallback으로 돌아가면 같은 회귀가 재발한다.
