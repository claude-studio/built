## 목표

Claude Code plugin 형태로 배포될 때 marketplace/plugin metadata, vendor license/notice, provider README와 smoke 문서가 package에서 누락되지 않도록 release 전 자동 검증 기준을 만든다.

## 구현 내용

- `scripts/check-plugin-release.js`와 `package.json`의 `npm run check:plugin-release`로 release 전 packaging guard를 추가했다.
- `.claude-plugin/plugin.json`, `.claude-plugin/marketplace.json`, `plugins/built/.claude-plugin/plugin.json`의 필수 metadata를 검증한다.
- `plugins/built` package 안에서 `README.md`, `docs/ops/provider-setup-guide.md`, `docs/smoke-testing.md`, provider helper script, provider skill, `vendor/codex-plugin-cc/LICENSE`, `vendor/codex-plugin-cc/NOTICE`가 해석되는지 확인한다.
- `plugins/built`를 임시 package root로 `dereference` 복사한 격리 snapshot에서도 같은 파일과 README 문서 링크, vendor 고지 내용이 유효한지 검증한다.
- `test/plugin-packaging.test.js`에 격리 package 복사 후 필수 파일과 README/docs/vendor 기준을 확인하는 regression test를 추가했다.
- `docs/ops/plugin-release-checklist.md`에 release 전 필수 명령, package 포함 기준, vendor LICENSE/NOTICE 기준, 수동 확인 항목을 정리했다.

## 결정 내용

- release guard는 repository root가 아니라 `plugins/built` package source와 격리 package snapshot을 기준으로 판단한다.
  사용자 설치는 repository root 전체가 아니라 marketplace source인 `./plugins/built`를 기준으로 이뤄지므로, root에 파일이 있어 통과하는 검증은 실제 배포 누락을 놓칠 수 있기 때문이다.
- 현재 package source의 symlink 구조는 유지하되, release 검증은 symlink를 `dereference` 복사한 snapshot에서도 통과해야 한다.
  개발 중에는 root와 package source drift를 줄이고, release 전에는 실제 package에 파일 내용이 포함되는지 확인하기 위한 절충이다.
- vendor license/notice 검증은 존재 여부만 보지 않고 Apache License 2.0, OpenAI copyright, notice 문구를 확인한다.
  빈 파일이나 잘못된 고지 파일이 package에 포함되어도 법적 고지 누락 위험은 해소되지 않기 때문이다.

## 재발 방지 포인트

- `plugins/built` package source 기준 검증 없이 repository root 파일만 확인하면 marketplace 설치 후 README, provider guide, smoke guide, vendor notice 누락을 놓칠 수 있다.

- vendor LICENSE/NOTICE는 파일 존재만으로 충분하지 않다.
  Apache License 2.0 전문과 OpenAI copyright/notice 문구를 함께 확인해야 한다.
