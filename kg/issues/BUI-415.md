## 목표

Report 결과가 다음 Plan의 prior art로 재사용될 수 있도록 agent folder 내부 Markdown KG writer와 문서 관리 구조를 구현한다.
KG 산출물은 target repo나 built plugin repo가 아니라 `~/Desktop/agents/codex-pdca-agent/projects/<project-slug>/kg/` 내부에만 생성되어야 한다.

## 구현 내용

- `src/agent-kg-writer.js`를 추가해 `kg/_index.md`, `kg/issues/`, `kg/decisions/`, `kg/patterns/`, `kg/entities/`, `kg/workflows/` Markdown writer를 구현했다.
- Report 단계에서 `kg_candidates`를 받아 issue 문서와 decision/pattern/entity/workflow 초안을 agent-local KG root 아래에 분리 생성하도록 연결했다.
- `scripts/report.js`와 `scripts/run.js`가 report artifact와 `root-context.json`에 `agent_kg_issue`, `agent_kg_root` 경로를 기록하도록 정렬했다.
- `skills/plan/SKILL.md`에 다음 Plan이 agent-local KG를 prior art로 읽는 경로를 추가했다.
- `skills/report/SKILL.md`에 Report 단계 KG draft 후보 작성 지침을 추가했다.
- `docs/contracts/file-contracts.md`와 `kg/workflows/root-artifact-contract-validation.md`의 구 `kg_draft`/`kg_draft_target_root` 계약을 `agent_kg_issue`/`agent_kg_root` 기준으로 갱신했다.
- `test/agent-kg-writer.test.js`, `test/report.test.js`, `test/run.test.js`로 writer와 report/run artifact 계약을 검증했다.

## 결정 내용

- Report KG draft 계약은 target repo `kg_draft`가 아니라 agent-local `agent_kg_issue`와 `agent_kg_root`로 표현한다.
- provider는 KG 파일을 직접 쓰지 않는다. Report runner가 provider 결과를 해석하고 writer 계층을 호출한다.
- agent-local KG root는 `~/Desktop/agents/codex-pdca-agent/projects/<project-slug>/kg/`다. target project root, built plugin repo, 설치형 plugin package/cache 아래 `kg/`는 agent-local KG artifact 저장소가 아니다.
- `kg/issues/BUI-385.md`에 남아 있는 구 `kg_draft`/`kg_draft_target_root` 계약은 BUI-415 이후 superseded 기록으로 취급한다. 새 계약의 기준은 `docs/contracts/file-contracts.md`와 `kg/decisions/report-agent-local-kg-contract.md`다.

## 결정 이유

- PDCA의 Report 결과가 다음 Plan으로 이어지려면 durable issue/decision/pattern/entity/workflow 기억이 agent runtime이 읽는 위치에 있어야 한다.
- target repo에 KG draft를 쓰면 제품 코드 저장소와 agent memory가 섞이고, built plugin repo나 cache에 쓰면 실제 프로젝트별 기억이 분리되지 않는다.
- provider가 직접 파일을 쓰면 provider별 산출물 계약이 갈라진다. 기존 provider/runner 경계처럼 파일 계약은 runner와 writer 계층이 소유해야 한다.
- `agent_kg_issue`와 `agent_kg_root`를 artifact metadata에 남기면 다음 Plan과 리뷰어가 전체 로그 없이 KG 위치를 추적할 수 있다.
