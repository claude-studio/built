## 목표

`/built:run-opus`와 `/built:run-sonnet`이 Claude permission request로 멈췄을 때
사용자가 실패 원인과 다음 행동을 바로 알 수 있게 한다. `/built:status <feature>`도 같은
failure code에 대해 구체적인 remediation 선택지를 보여야 한다.

## 구현 내용

- `src/providers/failure.js`에 `claude_permission_request` 전용 remediation 문구를 추가했다.
- `scripts/run.js`는 run 실패 후 `last_failure.code === "claude_permission_request"`이면
  일반 실패 출력 뒤 전용 안내를 추가로 출력한다.
- `scripts/status.js`는 같은 failure code에 대해 `remediation` 블록을 출력한다.
- `/built:run-opus`, `/built:run-sonnet`, `/built:run`, `/built:status` skill 문서에
  Codex 우회, 명시적 `.claude/settings.json` allow rule 추가, 수동 Claude permission 설정
  확인 선택지를 반영했다.
- 관련 테스트는 provider failure action 문구와 status remediation 출력 경로를 고정한다.

## 결정 사항

- 위험 권한을 자동 적용하지 않는다.
  특히 `--dangerously-skip-permissions`는 안내에서 자동 적용하지 않는다고 명시한다.
- Claude permission 문제의 권장 우회는 BUI-305에서 추가된 `/built:run-codex-do <feature>`로
  Do/Iter를 Codex 경로에서 재실행하는 방식으로 둔다.
- `.claude/settings.json` allow rule 추가는 사용자 승인과 명시적 범위가 있을 때만 가능한
  선택지로 안내한다.
- provider run-request schema나 Claude CLI permission flag의 정식 계약은 이번 범위에서
  확장하지 않는다.
