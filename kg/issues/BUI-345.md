## 목표

README와 provider config 계약에서 허용하는 `providers.do: "codex"` 문자열 단축형이 실제 실행 snapshot에서도 안전하게 동작하도록 맞춘다.
완료 기준은 README 예시 기반 Run 설정이 `do`/`iter`에서 Codex `read-only` sandbox 오류로 즉시 실패하지 않고, 잘못된 상세형 설정은 phase와 필요한 sandbox를 포함한 명확한 오류로 실패하는 것이다.

## 구현 내용

- `src/providers/config.js`에서 문자열 단축형 여부를 보존한 뒤 provider capability 정책의 phase별 기본 sandbox를 주입하게 했다.
- Codex 단축형은 `do`/`iter`에서 `workspace-write`, `plan_synthesis`/`check`/`report`에서 `read-only`로 정규화된다.
- 상세형에서 `sandbox: "read-only"`를 명시한 Codex `do`/`iter` 설정은 기존 검증대로 거부된다.
- `README.md`와 `docs/contracts/provider-config.md`에 문자열 단축형도 sandbox 기본값을 함께 정규화한다는 설명을 반영했다.
- `test/providers-config.test.js`와 `test/providers-capabilities.test.js`에 Codex 단축형의 phase별 sandbox 정규화 회귀 테스트를 추가했다.

## 결정 내용

- provider shorthand는 유지한다.
  README와 `.built/config.json`의 `default_run_profile.providers` 계약이 provider name 문자열을 기본 사용자 입력으로 삼고 있기 때문이다.
- shorthand에서 sandbox를 비워 두지 않고 capability registry의 phase별 기본값을 parser 정규화 단계에서 주입한다.
  실행 시점까지 권한 충돌을 미루면 사용자가 문서대로 설정했는데 Run 단계에서 실패하는 문제가 반복된다.
- 상세형은 사용자의 명시적 의도를 보존하므로 `read-only`를 자동 교정하지 않는다.
  잘못된 상세형은 phase와 필요한 `workspace-write`를 포함한 오류로 실패해야 한다.
- 새 ADR은 만들지 않았다.
  이번 변경은 `kg/decisions/provider-config-default-and-sandbox-policy.md`의 기존 sandbox 정책과 `docs/contracts/provider-config.md` 계약을 구현에 맞게 보완한 것이다.

## 재발 방지 포인트

- `providers.<phase>` 문자열 단축형을 바꿀 때는 provider name뿐 아니라 phase별 capability 기본값까지 snapshot 결과로 검증한다.
- `do`/`iter` Codex 경로는 `workspace-write`가 필요한 쓰기 phase라는 점을 parser 테스트로 고정한다.
- 상세형 설정은 자동 보정하지 않는다.
  사용자가 명시한 잘못된 sandbox는 runner 실행 전에 명확한 오류로 실패시켜야 한다.
- README 예시와 `docs/contracts/provider-config.md`가 shorthand를 허용하면 parser/normalization 테스트도 같은 예시를 따라야 한다.
