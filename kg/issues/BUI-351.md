## 목표

Plan/Design/Run dogfooding 실패 보고에서 어느 target project의 `.built/runs`, `.built/features`, `.built/runtime`을 봐야 하는지 바로 추적할 수 있게 한다.
plugin repo, target project, provider execution root, runtime root가 섞이는 실패 모드를 문서와 artifact evidence, doctor 검증으로 분리한다.

## 구현 내용

- `docs/contracts/file-contracts.md`에 `root-context.json` 계약을 추가하고 Plan, Design/`plan_synthesis`, Run phase별 `project_root`, `plugin_root`, `runtime_root`, `execution_root`, `result_root`, 주요 artifact 의미를 정리했다.
- `src/root-context.js`를 추가해 root context 생성, plugin root 추론, subpath warning, 로그 포맷을 공통화했다.
- `scripts/plan-draft.js`, `scripts/plan-synthesis.js`, `scripts/run.js`가 실행 시작 시 `root-context.json` 또는 시작 로그에 root/path 요약을 남기도록 했다.
- `src/providers/doctor.js`에 `root_separation` check를 추가해 feature가 지정된 상태에서 cwd가 plugin/repository root로 보이고 target feature spec이 없으면 hard failure로 처리한다.
- Reviewer FAIL 이후 `plan_synthesis`의 명시 `BUILT_RESULT_ROOT` 경로와 실제 `plan-synthesis.json`/`.md` writer 경로가 같도록 `writePlanSynthesisOutput`에 `resultRoot`를 전달했다.

## 결정 내용

- `project_root`는 target project 기준이며 plugin package/cache root를 target artifact 저장 기준으로 승격하지 않는다.
- `plugin_root`는 helper code 위치 추적용 metadata이며 `.built` artifact의 기준이 아니다.
- `runtime_root`는 lifecycle artifact 기준이고, 기본값은 `<project_root>/.built/runtime`이다.
- `result_root`는 사람이 읽는 phase 결과와 progress/log artifact 위치다. `plan_synthesis`에서 별도 `BUILT_RESULT_ROOT`를 주입하면 `root-context.json`, 시작 로그, canonical writer가 같은 root를 가리켜야 한다.
- `root_separation`은 target project와 plugin repo 혼동이 의심될 때 smoke 실행 전 hard failure로 막는다.

## 결정 이유

- dogfooding 실패 분석은 전체 터미널 로그보다 artifact 위치 evidence를 먼저 본다. 시작 시점 root-context가 없으면 실패 산출물이 target project에 있는지 plugin repo에 있는지 확인하는 비용이 커진다.
- plugin script는 절대 경로로 실행될 수 있고 provider execution cwd는 worktree일 수 있으므로 `process.cwd()`와 helper `__dirname` 중 하나만 SSOT로 삼으면 root drift를 놓친다.
- warning만으로는 target feature가 없는 plugin repo cwd 실행을 막기 어렵다. provider-doctor 단계에서 hard failure로 잡아야 잘못된 root에 smoke artifact를 쓰는 회귀를 줄일 수 있다.

## 재발 방지 포인트

- Plan/Design/Run artifact 경로를 바꾸면 `docs/contracts/file-contracts.md`의 `root-context.json` 표와 `src/root-context.js` field 의미를 함께 갱신한다.
- helper `__dirname` 또는 plugin root를 target project artifact 저장 기준으로 사용하지 않는다.
- `BUILT_RESULT_ROOT`나 명시 result root를 도입하는 phase는 root-context artifact path, stdout log, writer output path를 같은 값으로 검증한다.
- provider-doctor의 `root_separation` failure를 warn으로 낮추지 않는다. target feature가 없는 plugin repo cwd 실행은 smoke 전에 차단해야 한다.
