## 목표

2026-04-27 KST dogfooding 점검에서 확인된 Codex real Run 회귀를 수정한다.
`plan_synthesis`가 app-server JSON schema 오류로 실패했고, 같은 structured output 경로를 쓰는 `check` phase도 Codex provider에서 깨질 수 있었다.

완료 목표는 Codex app-server에 올바른 JSON Schema object를 전달하고, `plan_synthesis`와 `check`가 provider와 무관하게 `structuredOutput`을 받을 수 있게 하는 것이다.

## 구현 내용

- `src/pipeline-runner.js`에서 Codex 경로로 넘기는 schema를 `{ schema: ... }` wrapper로 감싸지 않고 JSON Schema 자체로 전달하게 했다.
- `src/providers/codex.js`가 string/object `outputSchema`를 JSON Schema object로 정규화하고, 잘못된 JSON이나 `{ schema: ... }` wrapper를 명확한 오류로 거부하게 했다.
- Codex structured output strict schema 조건을 provider 진입 전에 검증한다.
  object schema는 `additionalProperties: false`를 가져야 하고, 모든 `properties` key가 `required`에 포함되어야 한다.
- Codex 성공 결과의 최종 JSON text를 parse해 `structuredOutput`으로 반환하게 했다.
  이로써 `plan_synthesis`와 `check`는 provider별 text 반환 차이에 기대지 않는다.
- `scripts/check.js`의 schema를 Codex strict structured output 요구사항에 맞췄다.
- `test/providers-codex.test.js`와 `test/check.test.js`에 Codex app-server 전달 schema와 check schema 회귀 테스트를 추가했다.

## 결정 내용

- Codex app-server에는 `outputSchema` 값으로 JSON Schema object 자체를 전달한다.
  `{ schema: ... }` wrapper는 app-server가 `type: object` schema로 해석하지 못하므로 금지한다.
- 기존 caller 호환을 위해 string schema 입력은 provider 경계에서 parse하되, parse 실패나 strict schema 미충족은 조용히 fallback하지 않고 즉시 실패시킨다.
- `structuredOutput`은 Codex provider 반환 계약에 포함한다.
  phase script가 Codex raw text를 직접 parse하거나 provider별 분기를 늘리면 runner/control-plane 책임 경계가 흐려지기 때문이다.
- Codex check real smoke는 기본 `npm test`에 묶지 않고 opt-in/manual evidence로 유지한다.
  실제 Codex 경로는 인증, app-server, 로컬 runtime 상태에 의존한다.

## 재발 방지 포인트

- `runPipeline()`은 provider별 schema API 차이를 흡수해야 한다.
  phase script가 Codex app-server wrapper나 Claude `jsonSchema` 세부사항을 직접 알게 만들지 않는다.
- Codex structured output을 추가하는 새 phase는 app-server strict schema 조건을 먼저 만족해야 한다.
  object schema마다 `additionalProperties: false`와 모든 properties의 `required` 포함 여부를 테스트로 고정한다.
- `{ schema: ... }` wrapper는 Codex app-server 전달 계약으로 쓰지 않는다.
  해당 shape가 필요하면 runner 내부 입력 호환용으로만 해석하고 provider 전송 전에는 JSON Schema object로 정규화한다.
- real Codex smoke는 기본 테스트에 포함하지 않는다.
  대신 release-blocking provider 변경에서는 `provider-doctor`, fake/unit 테스트, opt-in real smoke evidence를 함께 남긴다.
- provider는 결과 파일을 직접 쓰지 않는다.
  Codex provider는 `structuredOutput`과 표준 event를 반환하고, `check-result.md`와 `plan-synthesis.*` 기록은 runner/helper가 맡는다.
