## 목표

provider log뿐 아니라 smoke artifact, provider comparison report, KG 후보, Telegram notification까지 private path와 secret 후보가 사용자-facing 산출물에 남지 않게 한다.
특히 local daemon path, Multica workspace path tail, workspace UUID, token, chat id 후보가 public summary와 artifact에 직접 노출되지 않도록 한다.

## 구현 내용

- `scripts/sanitize.js`의 `maskPrivatePaths()`가 `~/multica_workspaces/<workspace-id>/...`, macOS/Linux absolute workspace path 후보, Codex local daemon path 후보를 public placeholder로 치환하도록 보강되었다.
- `sanitizeText()`와 `sanitizeJson()` 재사용 경로를 유지해 Markdown, JSON artifact, writer output, notification text가 같은 redaction helper를 거치게 했다.
- `src/progress-writer.js`와 `src/providers/standard-writer.js`의 public artifact 저장 경로가 sanitize된 text/JSON writer를 사용한다는 회귀 테스트를 보강했다.
- `scripts/compare-providers.js`는 comparison report와 candidate artifact 저장 시 sanitize된 `writeText`/`writeJson` 경로를 사용한다.
- `scripts/notify.js` notification 문구가 token, chat id, private path 후보를 그대로 보내지 않도록 테스트를 추가했다.
- `docs/contracts/file-contracts.md`, `docs/contracts/smoke-artifact.md`, `docs/ops/provider-comparison-mode.md`에 public summary와 debug detail의 경계, private path 원문 금지 기준이 보강되었다.

## 결정 사항

- public summary, result markdown, notification, comparison report에는 raw provider stderr/stdout, `debug_detail`, token/chat id, workspace UUID, local daemon path, private workspace path를 넣지 않는다.
  사용자가 조치할 수 있는 안전한 요약만 public 계층에 둬야 issue/comment와 notification으로 재사용될 수 있기 때문이다.
- `failure.debug_detail`은 제거하지 않고 sanitize 후 디버그 전용 계층에만 둔다.
  운영자는 실패 원인 분석에 raw 맥락이 필요하지만, public summary로 승격되는 순간 secret과 private identifier 노출 위험이 커진다.
- private workspace path는 홈 경로 축약보다 강하게 `[REDACTED_WORKSPACE]`로 전체 치환한다.
  workspace UUID만 숨기고 `/workdir/built` 같은 tail을 남기면 내부 runtime 구조와 provider execution 위치가 노출될 수 있기 때문이다.
- 새 artifact writer나 provider-specific secret/path 형식이 추가되면 helper를 새로 만들기보다 `scripts/sanitize.js` fixture를 확장하고 기존 writer 경유를 검증한다.

## 재발 방지 포인트

- 새 public artifact writer를 추가하면 저장 직전 `sanitizeText()` 또는 `sanitizeJson()`을 통과시키고, token/chat id/private path fixture를 함께 추가한다.
- workspace path redaction은 UUID만 제거하는 방식으로 충분하지 않다.
  tail path가 남으면 runtime 위치와 repository context가 노출될 수 있으므로 전체 private workspace path를 치환한다.
- `debug_detail`은 public summary, notification, status/result markdown으로 승격하지 않는다.
  raw provider detail이 필요하면 sanitize된 로그 또는 디버그 전용 필드에서 확인한다.
- provider별 secret 형식이나 local daemon path 형식이 늘어나면 `scripts/sanitize.js` fixture와 public file contract 테스트를 동시에 확장한다.
- KG와 문서에는 raw execution dump, private environment value, workspace UUID, 실제 로컬 경로를 남기지 않는다.
