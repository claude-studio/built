## 목표

`iter` phase가 큰 feature spec, `do-result.md`, `check-result.md`를 그대로 prompt에 포함해 provider context limit에 가까워지는 문제를 막는다.
artifact 크기가 커져도 prompt 크기가 무제한으로 증가하지 않게 하고, check issue 해결에 필요한 핵심 feedback은 보존한다.

## 구현 내용

- `scripts/iter.js`에 `BUILT_ITER_PROMPT_MAX_CHARS`와 `BUILT_ITER_PROMPT_WARN_CHARS` 기반 prompt budget guard를 추가했다.
- feature spec, do-result, check-result 입력을 bounded excerpt로 축약하는 경로를 추가했다.
- `check-result.md`는 status/frontmatter, 수정 필요 항목, diff/error/hook 관련 줄을 우선 보존하도록 처리했다.
- provider context limit 근접 또는 초과 전에 prompt chars 로그, warning, 사용자-facing 실패 진단, state 실패 사유를 남기도록 했다.
- `test/iter.test.js`에 큰 do-result/check-result fixture 회귀 테스트를 추가했다.
