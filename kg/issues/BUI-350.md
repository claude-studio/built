## 목표

`scripts/do.js`가 `kg/issues`, `kg/decisions`, `kg/workflows`의 전체 본문을 계속 prompt에 붙여 repo history 증가에 따라 Do prompt가 선형으로 커지는 문제를 줄인다.
관련 KG context는 유지하되 provider context limit, 비용, latency 위험을 control plane에서 미리 낮춘다.

## 구현 내용

- Do phase KG context 생성을 `scripts/do-kg-context.js` helper로 분리했다.
- `kg/goals/north-star.md`와 핵심 workflow는 항상 포함하고, feature spec, `plan-synthesis.json`, run request/provider 설정에서 추출한 BUI 번호, 파일 경로, provider/phase 키워드, frontmatter `tags`, `keywords`, `context_issue`, `kg_files`로 관련 KG를 선별한다.
- `kg/issues`, `kg/decisions`, `kg/workflows` 전체 본문을 무조건 append하지 않고 full body 문서 수와 index 문서 수를 상수 제한으로 묶었다.
- `BUILT_DO_PROMPT_MAX_CHARS`, `BUILT_DO_PROMPT_WARN_CHARS` 문자 budget guard를 추가해 budget 초과를 provider 실행 전 경고 또는 오류로 처리한다.
- provider context 초과 메시지를 `model_response` + `provider_context_limit_exceeded`, `retryable=false`, `blocked=true`로 분류하도록 `src/providers/failure.js`를 보강했다.

## 결정 내용

- Do phase KG context 제한은 provider adapter가 아니라 `scripts/do.js` control plane에서 처리한다.
- provider별 context limit table을 hardcode하지 않고 문자 budget과 bounded selector를 기본 방어선으로 둔다.
- 비관련 KG는 본문 대신 `id`, `title`, `status`, `tags`, `keywords`, `summary` 수준의 index로 축약한다.
- context overflow는 provider failure taxonomy 안에서 `model_response` kind의 `provider_context_limit_exceeded` code로 수렴한다.

## 결정 이유

- provider adapter가 KG 선별을 담당하면 provider별 prompt 구성과 file/event contract가 갈라질 수 있다.
- provider context limit은 모델과 설정에 따라 변하므로 hardcoded token table보다 control plane의 conservative character budget이 안정적인 1차 guard다.
- BUI-43 당시 전체 KG 포함은 KG가 소규모라는 전제에 기반했지만, issue/decision 문서가 증가하면서 전제가 깨졌다.
- context overflow를 opaque provider error로 두면 사용자가 KG budget 조정이 필요한지 provider 장애인지 구분하기 어렵다.

## 재발 방지 포인트

- `scripts/do.js`가 `kg/issues`, `kg/decisions`, `kg/workflows` glob 결과 전체 본문을 직접 append하는 경로를 다시 만들지 않는다.
- KG selector를 바꿀 때는 관련 BUI/ADR 포함, 비관련 KG bounded index, budget warning/error 회귀 테스트를 함께 갱신한다.
- context overflow는 provider별 raw message가 아니라 공통 failure taxonomy의 `provider_context_limit_exceeded`로 분류한다.
- provider별 token limit table을 추가하기 전에 control plane budget과 selector signal로 해결 가능한지 먼저 검토한다.
