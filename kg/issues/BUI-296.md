## 목표

`/built:plan` Phase 5 Save에서 `scripts/plan-save.js`가 feature spec에 이미 포함된 보조 문서 컨텍스트를 잃지 않게 한다.
사용자가 plan 인터뷰에서 답한 엔티티 설명, 아키텍처 결정 이유, tradeoff, 거부된 대안, reference pattern 설명이 신규 보조 문서에 반영되어야 했다.

## 구현 내용

- `saveAuxDocs()`가 feature spec 본문을 보조 문서 builder에 전달하도록 수정했다.
- `entities/*` 보조 문서는 `Content & Data > Entities`의 해당 wikilink 라인 설명을 신규 문서 본문과 feature 사용 설명에 보존한다.
- `decisions/*` 보조 문서는 `Architecture`의 채택 설명, `Tradeoffs`, 선택하지 않은 대안을 신규 문서에 보존한다.
- `patterns/*` 보조 문서는 `Build Plan > Reference Patterns`의 해당 wikilink 라인 설명을 신규 문서에 보존한다.
- 관련 설명이 없는 wikilink는 기존 `내용을 채워주세요` 스켈레톤 fallback으로 생성한다.
- 기존 보조 문서는 overwrite하지 않는 멱등성을 유지했다.
- `test/plan-save.test.js`에 도그푸딩 재현 케이스와 overwrite 멱등성 회귀 테스트를 추가했다.

## 결정 사항

- feature spec을 Phase 5 보조 문서 생성의 SSOT 컨텍스트로 사용한다.
  Plan 단계에서 이미 답한 내용을 다시 쓰게 만들지 않는 것이 `/built:plan` handoff의 핵심 가치이기 때문이다.
- 보조 문서 생성은 deterministic extraction만 수행하고 LLM 재작성은 호출하지 않는다.
  Save 단계가 환경과 provider 상태에 의존하지 않아야 하며, 원문 컨텍스트 보존이 이번 결함의 핵심 해결책이기 때문이다.
- 기존 보조 문서 overwrite 금지는 유지한다.
  생성기는 비어 있는 지식 레이어를 채우는 bootstrap 역할이며, 사용자가 이후 편집한 보조 문서를 feature spec 원문으로 되돌리면 안 되기 때문이다.

## 재발 방지 포인트

- Phase 5 Save에서 보조 문서를 새로 만들 때는 feature spec의 해당 section과 wikilink 인라인 설명을 먼저 추출한다.
  정보가 없을 때만 빈 스켈레톤 fallback을 사용한다.
- `entities`, `decisions`, `patterns`는 같은 wikilink 문법을 쓰더라도 추출 기준 section과 보존해야 하는 주변 컨텍스트가 다르다.
  하나의 범용 라인 추출만으로 decision tradeoff나 거부된 대안을 잃지 않게 한다.
- 기존 보조 문서가 있으면 생성기가 덮어쓰지 않는다.
  사용자가 보조 문서를 편집한 뒤 `/built:plan` 저장을 다시 실행하는 경로를 회귀 테스트로 유지한다.
- 문서 스키마나 Phase 5 heading을 바꾸는 후속 변경은 `test/plan-save.test.js`의 context extraction 케이스도 함께 갱신한다.
