## 목표

execution worktree cwd에서 `plan_synthesis`가 실행될 때 `run-request.json.planPath`의 absolute/relative 표현 방식에 따라 root spec과 worktree spec을 혼용할 수 있는 경로를 제거한다.
같은 feature의 Design 입력은 항상 같은 feature spec source of truth를 읽어야 하며, artifact만 보고 어떤 spec이 사용됐는지 확인할 수 있어야 한다.

## 구현 내용

- `scripts/plan-synthesis.js`가 payload 생성 시 `BUILT_PROJECT_ROOT`를 `featureSpecRoot`로 전달하고 `featureSpecSource`를 `control_root`로 고정했다.
- `src/plan-synthesis.js`가 relative `planPath`와 execution worktree 내부 absolute `planPath`를 control root 기준의 같은 상대 feature spec으로 정규화하도록 보강했다.
- `feature_spec_source` 메타데이터를 plan synthesis payload, `plan-synthesis.json`, markdown front matter에 기록했다.
- `src/root-context.js`가 `root-context.json`과 formatted root-context 출력에 `feature_spec_source`를 포함하도록 확장됐다.
- `docs/contracts/plan-synthesis-input.md`와 `docs/contracts/file-contracts.md`에 control root 기준과 `source`, `source_root`, `requested_path`, `resolved_path` 계약이 추가됐다.
- `test/plan-synthesis.test.js`에 absolute/relative planPath 일치, worktree relative drift 방지, worktree absolute drift 방지, root-context 기록 테스트가 추가됐다.

## 결정 내용

- Design / `plan_synthesis`의 feature spec source of truth는 control root다.
- `/built:run`이 execution worktree cwd에서 실행돼도 `planPath`는 control root 기준으로 정규화한다.
- artifact에는 `feature_spec_source.source`, `source_root`, `requested_path`, `resolved_path`를 남긴다.
- control root와 execution worktree 어느 쪽에도 속하지 않는 임의 absolute `planPath`는 기존 동작처럼 explicit absolute path로 해석한다.

## 결정 이유

- control root spec은 사용자가 승인한 Plan/Design 입력이며, execution worktree drift가 설계 기준을 바꾸면 안 된다.
- `planPath` 문자열 표현만 바꿨는데 다른 spec을 읽는 것은 downstream `do` phase 입력 안정성을 깨뜨린다.
- root-context는 `project_root`, `execution_root`, `result_root`를 구분하므로, spec source도 별도 감사 필드로 남겨야 실패 분석이 가능하다.
- provider boundary상 provider가 파일을 직접 쓰지 않고 runner/helper가 canonical artifact를 기록하는 구조를 유지해야 한다.

## 재발 방지 포인트

- `plan_synthesis` 입력 생성 경로를 바꿀 때는 `featureSpecRoot`가 control root로 유지되는지 확인한다.
- `planPath` 정규화 변경은 relative, control root absolute, execution worktree absolute, 외부 absolute path 케이스를 분리해 테스트한다.
- `root-context.json`과 `plan-synthesis.json`의 `feature_spec_source.resolved_path`가 같은 spec을 가리키는지 artifact 계약 테스트로 확인한다.
- worktree는 implementation isolation을 위한 실행 공간이며, Design feature spec source를 암묵적으로 바꾸는 기준으로 사용하지 않는다.
