## 목표

사용자가 수동으로 `run-request.json`을 편집하지 않아도 대표 provider 조합을 안전하게 적용할 수 있게 한다.
동시에 `skills/*/SKILL.md`에 남아 있던 Claude 전용 표현과 수동 JSON 작성 예시를 provider 구조와 실제 helper 동작에 맞춰 정리한다.

## 구현 내용

- `src/providers/presets.js`를 추가해 provider preset 정의, preset 조회, `run-request.json` 객체 생성, 파일 기록 API를 분리했다.
- `scripts/provider-preset.js` CLI helper를 추가해 `<feature> --preset <name> [--model <model>]` 형태로 `.built/runtime/runs/<feature>/run-request.json`을 생성할 수 있게 했다.
- 기본 preset은 `claude-default`, `codex-do`, `codex-plan`, `codex-all` 4개로 고정했다.
- helper는 `parseProviderConfig`를 통해 provider 이름, phase별 sandbox, 상세 설정을 검증한 뒤 파일을 쓴다.
- `test/providers-presets.test.js`를 추가해 preset 목록, 잘못된 preset, 직접 providers 검증, parser fixture, 파일 생성, `.built/config.json` 미접촉을 검증했다.
- `skills/do`, `skills/check`, `skills/iter`, `skills/report`, `skills/run`, `skills/run-opus`, `skills/run-sonnet` 문서를 provider 중립 표현과 helper 사용 흐름에 맞춰 정리했다.

## 결정 내용

- provider preset helper의 쓰기 대상은 `.built/runtime/runs/<feature>/run-request.json` 하나로 제한한다.
  `.built/config.json`은 built 전역 설정 검증 대상이며 provider phase 선택 위치가 아니다.
- `claude-default` preset은 `providers` 필드를 쓰지 않는 기존 Claude 기본 경로와 동등하게 둔다.
  기본 provider를 Codex로 바꾸지 않고 phase별 opt-in만 제공한다.
- `codex-do`는 `do`/`iter`를 Codex `workspace-write`로, `check`/`report`를 Claude로 둔다.
  구현과 검토 provider를 분리하는 대표 교차 검증 패턴이다.
- `codex-plan`은 `plan_synthesis`만 Codex `read-only`로 둔다.
  나머지 phase는 Claude fallback을 사용한다.
- `codex-all`은 모든 구현된 phase를 Codex로 지정하되, 파일을 쓰는 `do`/`iter`만 `workspace-write` sandbox를 사용한다.
- skill 문서에서는 사용자-facing 안내를 "설정된 provider" 중심으로 적고, `claude -p`는 Claude provider 구현 디테일이 필요한 곳에만 남긴다.

## 결정 이유

- 사용자가 JSON 구조를 직접 편집하면 provider 이름, sandbox, phase 키 오류가 늦게 발견될 수 있다.
  helper가 기존 parser를 재사용하면 실행 전 즉시 실패하고 parser 계약과 문서 예시가 함께 고정된다.
- provider 선택을 `.built/config.json`으로 옮기면 전역 기본값처럼 보이고 기존 config 검증 계약과 충돌한다.
  run 단위 요청 파일에만 쓰면 feature별 실험과 rollback이 단순하다.
- `run-opus`/`run-sonnet`은 Claude provider 모델 preset으로 유지해야 한다.
  provider preset과 모델 preset을 섞지 않으면 Codex opt-in과 Claude 모델 선택이 서로 덮어쓰지 않는다.

## 재발 방지 포인트

- provider preset을 추가할 때는 `src/providers/presets.js`의 preset 정의와 `test/providers-presets.test.js` parser fixture를 함께 갱신한다.
- 파일을 쓰는 Codex phase인 `do`와 `iter`에는 `sandbox: "workspace-write"`를 명시한다.
  `read-only`는 parser 검증 단계에서 실패해야 한다.
- `claude-default` preset은 빈 `providers` 객체를 유지해 기존 요청 파일과 동일한 Claude fallback 의미를 보존한다.
- skill 문서에서 provider 실행을 설명할 때는 provider-neutral 산출물 계약과 subprocess 구현 디테일을 분리한다.
- helper 문서나 예시가 `.built/config.json`에 `providers`를 쓰도록 안내하면 안 된다.
