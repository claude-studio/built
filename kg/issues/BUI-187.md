## 목표

이슈 ID 혼선, 오래된 execution, 중복 assign 때문에 현재 canonical routing과 다른 실행이 시작돼도
코드 변경, commit, push, PR 생성, issue status 변경을 만들지 않도록 운영 정책과 agent KG를
정리한다.

## 구현 내용

- `docs/ops/misdirected-execution-policy.md`에 status, assignee, 최신 routing/correction comment,
  선행조건, canonical PR 확인을 작업 계속 여부의 gate로 정의했다.
- `kg/agents/builder.md`에 시작 전, 코드 수정 전, 테스트 전, PR 생성 전 재확인 절차를 추가했다.
- `kg/agents/coordinator.md`에 잘못 시작된 execution을 직접 cancel할 수 없을 때 최신 canonical
  routing comment로 안전하게 무시되도록 하는 기준을 보강했다.
- `kg/agents/operator.md`에 stale running execution을 정상 canonical 실행과 분리해 무시/정리하는
  절차를 정리했다.
- `kg/workflows/misdirected-execution-safety.md`를 추가해 Builder, Coordinator, Operator가
  재사용할 수 있는 중단/무시 워크플로우를 남겼다.
- `kg/issues/BUI-187.md`에는 PR branch mapping 스켈레톤을 만들고, Recorder 단계에서 본 기록과
  durable KG 파일 목록을 완성했다.

## 결정 내용

- 현재 issue의 `status`, `assignee`, 최신 routing/correction comment를 execution 계속 여부의
  SSOT로 삼는다.
- 조건이 맞지 않는 실행은 코드 수정, commit, push, PR 생성, status 변경, 다른 정상 실행 중단을
  하지 않는다.
- 잘못 시작된 실행은 한글과 KST 기준으로 현재 canonical 상태, 무시 사유, 로컬/원격 변경 없음,
  후속 조치를 코멘트로 남기고 종료한다.
- PR 생성 직전에는 같은 BUI 번호의 open PR과 현재 head branch의 open PR을 확인한다. 기존
  canonical PR이 있으면 새 PR을 만들지 않고 해당 branch를 이어 쓴다.
- handoff/result comment는 status와 assignee 변경보다 먼저 남겨 audit trail이 끊기지 않게 한다.

## 결정 이유

- Multica CLI에 running execution을 직접 취소하는 명령이 없는 상황에서는 오래된 실행을 제거하는
  것보다 현재 canonical routing과 맞지 않는 실행이 산출물을 만들지 못하게 하는 gate가 더 안전하다.
- status와 assignee는 플랫폼이 이미 제공하는 현재 routing 상태이고, 최신 comment는 correction,
  prerequisite, PR/branch 정보를 담는 handoff SSOT다.
- PR 생성 직전 재확인을 추가하면 오래된 실행이 뒤늦게 새 branch나 중복 PR을 만드는 회귀를 막을 수
  있다.
- 정상 진행 중인 다른 execution과 open PR을 건드리지 않는 것이 운영 피해를 최소화한다.

## 재발 방지 포인트

- 모든 agent는 작업 시작 전 `issue get`과 최신 handoff/result/correction comment를 확인한다.
- 현재 assignee가 아니거나 issue가 현재 역할의 진행 상태가 아니면 작업 결과를 만들지 않는다.
- backlog로 되돌아간 issue에 남은 running execution은 새 작업 신호로 보지 않는다.
- PR 생성 전에는 같은 BUI 번호와 현재 head branch 기준 open PR을 확인한다.
- canonical PR이 불명확하면 main direct commit이나 새 PR 생성 없이 Coordinator 판단을 요청한다.
- handoff comment에는 다음 역할이 raw execution log를 읽지 않아도 PR URL, branch, head commit,
  검증 결과, blocker, KG 후보를 판단할 수 있게 남긴다.
