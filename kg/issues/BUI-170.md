## 목표

provider 관련 테스트가 늘어난 상태에서 운영자가 회귀 범위를 빠르게 좁힐 수 있도록 테스트 명령을 세분화한다.
기본 `npm test`, provider offline 테스트, fake provider E2E, comparison mode 테스트, real Codex smoke의 책임 경계를 명확히 나누고 CI-ready 명령 목록을 문서화한다.

## 구현 내용

- `package.json`에 provider 전용 npm script를 추가했다.
  - `npm run test:provider`: provider 단위 테스트 그룹과 provider fake E2E를 함께 실행한다.
  - `npm run test:provider:unit`: provider 단위 테스트 그룹만 실행한다.
  - `npm run test:provider:contracts`: `test/file-contracts.test.js` 단독 실행이다.
  - `npm run test:provider:e2e`: provider 관련 fake E2E 시나리오 04, 05만 실행한다.
  - `npm run test:provider:compare`: `test/compare-providers.test.js` 단독 실행이다.
- `scripts/run-tests.js`에 `--provider` 플래그를 추가해 provider 단위 테스트 그룹을 선택 실행할 수 있게 했다.
- `test/e2e/e2e-runner.js`에 `--filter <token>` 플래그를 추가해 파일명 기준으로 E2E 시나리오를 좁힐 수 있게 했다.
  `--filter provider`는 `04-fake-provider-file-contracts`, `05-provider-equivalence-contracts`를 선택한다.
- `docs/smoke-testing.md`에 basic/offline/provider/smoke 명령 계층을 정리했다.
  `npm test`와 `npm run test:provider*`는 CI-ready offline 명령으로, `npm run test:smoke:codex*`는 opt-in 전용 real smoke로 분리했다.

## 결정 사항

- provider 테스트 그룹은 `providers-*.test.js`, `provider-doctor.test.js`, `file-contracts.test.js`, `compare-providers.test.js`로 정의한다.
  provider adapter, diagnostics, file contract, comparison parser/runner 회귀를 한 번에 좁히는 운영 단위가 필요하기 때문이다.
- provider E2E 선택은 별도 manifest가 아니라 파일명 필터 `provider`를 사용한다.
  현재 시나리오 04, 05가 provider file contract와 equivalence contract를 명확히 나타내므로 단순한 additive runner 확장으로 충분하다.
- `npm run test:provider`는 offline CI-ready 명령으로 유지하고 real Codex smoke를 포함하지 않는다.
  real smoke는 인증, 네트워크, 비용, 로컬 Codex app-server 상태에 의존하므로 기본 회귀 신호와 섞으면 안 된다.
- comparison mode 회귀 기준은 `compare-providers.test.js`와 provider E2E 시나리오 04, 05를 함께 본다.
  단위 테스트는 parser/runner 계약을, fake E2E는 file contract와 provider equivalence golden fixture를 방어한다.

## 재발 방지 포인트

- `npm test`와 `npm run test:provider*`에는 real provider smoke를 추가하지 않는다.
  provider smoke는 `npm run test:smoke:codex*` 또는 명시적 환경 변수 opt-in으로만 실행한다.
- provider 단위 테스트 그룹을 바꿀 때는 `scripts/run-tests.js`의 `collectProviderUnitTests()` 설명, `docs/smoke-testing.md`의 포함 범위, npm script 이름을 함께 갱신한다.
- provider E2E 시나리오 파일명을 바꾸면 `--filter provider`가 여전히 시나리오 04, 05를 선택하는지 확인한다.
- comparison mode 회귀를 provider group에서 제외하지 않는다.
  `compare-providers.test.js`는 CI-ready provider 검증의 일부이고, 실제 provider 비교 smoke는 별도 opt-in으로 둔다.
- 테스트 출력 이름은 어떤 계약이 깨졌는지 추적 가능해야 한다.
  file contract, provider E2E, comparison mode를 뭉뚱그려 하나의 generic test 실패로만 남기지 않는다.
