## 목표

Codex `createStandardWriter` 경로가 Claude provider와 같은 `.built/features/<feature>/logs/<phase>.jsonl` 계약을 만족하도록 보강한다.
Run 실패나 dogfooding 실패 시에도 provider별 phase 이벤트, terminal 실패 이벤트, structured output 흐름을 같은 위치와 형식으로 추적할 수 있어야 한다.

## 구현 내용

- `src/providers/standard-writer.js`가 Codex/fake 표준 provider event를 `logs/<phase>.jsonl`에 redaction 후 append하도록 보강했다.
- 성공 run은 `phase_end`까지, 실패 run은 terminal `error`까지 phase log에 남도록 했다.
- `test/file-contracts.test.js`에 standard writer의 성공/실패 phase log artifact presence 검증을 추가했다.
- `test/e2e/scenarios/04-fake-provider-file-contracts.js`에 fake Claude/Codex provider artifact presence 검증을 추가했다.
- `docs/contracts/file-contracts.md`에 Claude/Codex/fake provider별 runtime artifact 차이와 동일성 기준을 명시했다.

## 결정 내용

- Codex와 fake provider의 phase log는 provider가 직접 파일을 쓰지 않고 `standard-writer`가 같은 위치에 append한다.
- phase log 동일성 기준은 raw payload shape가 아니라 경로, JSONL one-object-per-line 형식, redaction 적용, terminal `phase_end` 또는 `error` 보존이다.
- Claude legacy `progress-writer`와 Codex/fake `standard-writer` 경로는 payload 세부 필드가 달라도 같은 runtime artifact 계약을 만족해야 한다.
- 이번 변경은 신규 provider architecture decision이 아니라 ADR-5의 provider event normalize와 standard writer 계약, ADR-6의 provider 파일 책임 경계를 실행 artifact까지 보강한 기록이다.

## 결정 이유

- `progress.json`은 최신 snapshot이므로 실패 원인 분석과 structured output 흐름 재현에는 phase별 append 로그가 필요하다.
- provider별 writer 또는 provider 직접 append를 허용하면 파일 위치와 보안 redaction 경계가 provider마다 갈라질 수 있다.
- Codex app-server notification raw shape는 바뀔 수 있으므로 동일성 기준을 raw shape에 두면 불필요한 회귀와 문서 drift가 생긴다.
- terminal `error` 보존은 실패 run의 최종 상태와 재현 단서를 phase log에 남기기 위한 최소 계약이다.

## 재발 방지 포인트

- `standard-writer` 또는 provider event normalize를 바꿀 때는 성공 `phase_end`와 실패 terminal `error`의 `logs/<phase>.jsonl` presence 테스트를 함께 유지한다.
- 새 provider를 추가할 때는 provider가 built artifact를 직접 쓰지 않고 표준 이벤트를 writer에 넘기는지 확인한다.
- phase log 계약을 raw provider payload shape에 묶지 않는다. raw shape 변경은 normalize fixture와 adapter mapping 갱신으로 처리한다.
- 사용자-facing artifact에는 provider debug detail, local daemon path, workspace UUID, token 후보를 승격하지 않는다.
