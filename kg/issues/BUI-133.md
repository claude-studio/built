## 목표

Multica daemon host의 bare clone cache와 worktree를 사용하는 실행 환경에서 PR merge 후 작업 branch와 worktree가 stale 상태로 남지 않도록 정리 책임과 감지 기준을 문서화한다.
daemon-side worktree가 로컬 `git worktree list`만으로 보이지 않는 제약을 운영 절차에 반영하고, cleanup이 위험할 때 자동 삭제하지 않는 기준을 만든다.

## 구현 내용

- `docs/ops/worktree-cleanup-policy.md`를 추가해 daemon worktree 가시성, stale branch 감지 기준, cleanup 안전 규칙, Finisher/Operator 책임 경계, cleanup evidence 형식을 정리했다.
- `scripts/check-stale-branches.js`를 추가해 원격 `agent/` branch 기준으로 stale 후보와 blocked 후보를 감지한다.
  `--json` 출력과 `--remote <remote>` 옵션을 제공하며, 자동 삭제는 수행하지 않는다.
- `kg/agents/finisher.md`에 merge 후 원격 branch 삭제, `cleanup.js` 실행, cleanup evidence 기록 책임을 추가했다.
- `kg/agents/operator.md`에 stale branch/worktree 점검, `check-stale-branches.js` 사용, 안전 조건 미충족 시 blocked/알림 처리 책임을 추가했다.

## 결정 내용

- PR merge 직후 branch/worktree cleanup은 Finisher 책임으로 확정했다.
- 주기적 stale branch/worktree 점검은 Operator 책임으로 확정했다.
- daemon worktree 여부는 로컬 worktree 목록이 아니라 원격 branch 상태를 기준으로 판단한다.
- open PR, running 작업, unmerged commit, uncommitted 변경이 있으면 자동 삭제하지 않고 blocked/알림으로 남긴다.
- `scripts/check-stale-branches.js`는 감지 전용 smoke/운영 체크로 유지하고, 삭제는 명시적 후속 절차로 분리한다.

## 결정 이유

Finisher는 PR merge 시점의 canonical branch와 최종 상태를 알고 있으므로 merge 직후 cleanup evidence를 남기기에 적합하다.
Operator는 운영 점검에서 누락된 stale branch를 주기적으로 찾아야 하지만, PR merge나 최종 done 판단을 대신하지 않는다.

Multica daemon이 생성한 worktree는 사용자 로컬 프로젝트의 `git worktree list`에 나타나지 않으므로, cleanup 판단을 로컬 worktree 목록에 의존하면 stale branch를 놓칠 수 있다.
원격 branch 기준 점검은 daemon 환경과 로컬 환경 사이의 가시성 차이를 줄이는 보수적 기준이다.

## 재발 방지 포인트

- daemon-side worktree 정리 여부를 로컬 `git worktree list` 결과만으로 판단하지 않는다.
- cleanup 대상은 원격 `agent/` branch, merge 여부, PR 상태를 함께 확인한다.
- `gh` CLI가 없거나 PR 상태가 `unknown`이면 open PR이 없다고 가정하지 말고 수동 확인을 요구한다.
- open PR 또는 `origin/main`에 없는 커밋이 있으면 branch 삭제와 `cleanup.js` 실행을 중단한다.
- Finisher와 Operator는 cleanup evidence에 삭제한 branch, cleanup 결과, stale/blocked 개수, KST 실행 시각을 남긴다.
- cleanup 정책이 바뀌면 `docs/ops/worktree-cleanup-policy.md`, `kg/agents/finisher.md`, `kg/agents/operator.md`, `kg/workflows/daemon-worktree-cleanup.md`를 함께 갱신한다.
