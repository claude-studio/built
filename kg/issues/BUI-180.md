## 목표

provider 비교, smoke, broker lifecycle 운영으로 `.built/runtime` 및 임시 smoke 디렉토리에 쌓이는 run/comparison/smoke artifact의 보존 기준을 문서화하고, 안전한 cleanup 후보 점검 명령을 제공한다.

## 구현 내용

- `scripts/cleanup-artifacts.js`를 추가해 provider comparison worktree/branch와 smoke 임시 디렉토리 cleanup 후보를 점검한다.
- `--dry-run` 모드와 실제 삭제 모드를 분리했다.
- `--feature <name>`로 특정 feature의 comparison 후보만 제한해 확인할 수 있게 했다.
- `--smoke` 옵션으로 `/tmp/built-codex-*-smoke-*/` 잔여 디렉토리까지 함께 점검할 수 있게 했다.
- `docs/ops/artifact-retention-policy.md`를 추가해 run artifact, comparison evidence, smoke 임시 디렉토리의 보존/삭제 기준과 Finisher/Operator 책임 경계를 정리했다.
- `test/cleanup-artifacts.test.js`를 추가해 cleanup 후보 분류, blocked 조건, smoke TTL, dry-run/삭제 분리를 검증했다.

## 결정 내용

- comparison evidence directory(`report.md`, `diff.patch`, `providers/<candidate-id>/`, `manifest.json`, 로그)는 cleanup 대상으로 삼지 않고 audit evidence로 보존한다.
- cleanup 자동화 대상은 comparison candidate worktree와 compare branch, 그리고 24시간을 넘긴 smoke 임시 디렉토리로 제한한다.
- comparison worktree/branch 삭제는 open PR 없음, branch merge 완료 또는 PR closed/merged, uncommitted 변경 없음 조건을 모두 만족할 때만 허용한다.
- `BUILT_KEEP_SMOKE_DIR=1`로 남긴 smoke 임시 디렉토리는 디버그 기간 24시간을 넘긴 경우에만 삭제 후보로 본다.
- gh CLI로 PR 상태를 확인할 수 없으면 경고를 남기되, merge 여부와 open PR 보호 조건을 보수적으로 적용한다.

## 결정 이유

- provider comparison의 핵심 산출물은 사람이 품질과 acceptance criteria 충족을 판단하기 위한 evidence이므로, 저장 공간 정리 때문에 삭제되면 north-star의 관측 가능성과 복구 가능성을 해친다.
- worktree와 branch는 evidence 자체가 아니라 실행 환경 잔여물이므로 안전 조건을 만족하면 운영 부채를 줄이기 위해 정리할 수 있다.
- dry-run을 기본 운영 진입점으로 두면 Operator/Finisher가 삭제 전 후보와 blocked 사유를 이슈 코멘트에 남길 수 있다.
- smoke 임시 디렉토리는 기본적으로 자동 삭제되며, 남아 있는 경우는 디버그 목적이므로 짧은 보존 기간을 둔다.

## 재발 방지 포인트

- comparison evidence directory는 storage cleanup 명령으로 통째로 삭제하지 않는다.
- `node scripts/cleanup-artifacts.js`는 먼저 `--dry-run`으로 실행해 삭제 후보와 blocked 사유를 이슈 코멘트에 남긴 뒤 실제 삭제를 판단한다.
- open PR, unmerged branch, uncommitted 변경이 있으면 worktree/branch cleanup을 중단한다.
- smoke 임시 디렉토리는 `BUILT_KEEP_SMOKE_DIR=1`로 남긴 디버그 산출물만 cleanup 후보가 되며, 24시간 이내에는 보존한다.
- Finisher는 PR merge 직후 feature cleanup evidence를 남기고, Operator는 주기 점검에서 comparison/smoke cleanup 후보와 blocked 후보를 분리한다.
