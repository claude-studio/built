## 목표

real Codex smoke는 opt-in으로 분리되어 있지만 실행 결과가 일관된 산출물로 남지 않으면 실패 원인 비교와 재현이 어렵다.
Codex plan/do smoke 실행 또는 skip 결과를 `.built/runtime/smoke/<id>/summary.json`에 구조화해 남기고, 사용자에게 공유 가능한 짧은 한글 실패 요약과 민감정보 redaction 기준을 함께 고정한다.

## 구현 내용

- `scripts/smoke-artifact.js`를 추가해 smoke summary artifact 생성, 저장, 한글 실패 요약 포맷을 공통화했다.
- `scripts/smoke-codex-plan-synthesis.js`와 `scripts/smoke-codex-do.js`의 모든 종료 경로에 artifact 저장을 통합했다.
  opt-in 환경 변수가 없어 skip되는 경우에도 `skipped: true`, `success: true`로 요약을 남긴다.
- artifact schema를 `docs/contracts/smoke-artifact.md`에 문서화했다.
  저장 경로, `schema_version: 1.0.0`, 필수 필드, phase별 `verification`, failure taxonomy, redaction 정책을 포함한다.
- `.gitignore`에 `.built/runtime/smoke/`를 추가해 local runtime evidence가 버전 관리에 섞이지 않게 했다.
- `test/smoke-artifact.test.js`를 추가해 artifact 생성/저장, skip/success/failure 분기, 한글 failure summary, secret redaction을 검증했다.
- `docs/smoke-testing.md`에 smoke artifact 위치와 실패 공유 요약 절차를 추가했다.

## 결정 사항

- artifact 저장 경로는 `.built/runtime/smoke/<id>/summary.json`으로 고정한다.
  real smoke는 로컬 인증, 네트워크, 비용, provider availability에 의존하므로 외부 telemetry로 보내지 않고 로컬 evidence로 남기는 것이 현재 운영 경계에 맞다.
- schema는 `schema_version: 1.0.0`을 포함하고 하위 호환이 깨지는 변경 시 major를 올린다.
  smoke 결과를 issue/comment와 KG에서 장기 비교할 수 있어야 하므로 암묵적 JSON 구조보다 명시 버전이 필요하다.
- failure taxonomy는 `provider_unavailable`, `app_server`, `auth`, `sandbox`, `timeout`, `model_response`, `unknown` 7종으로 기록한다.
  기존 smoke 실패 원인축을 artifact와 사용자 메시지에 동일하게 반영해 운영자가 실패를 빠르게 분류할 수 있게 하기 위함이다.
- artifact 저장 전 `scripts/sanitize.js`의 `sanitizeJson()`을 재사용한다.
  API key, GitHub token, 홈 경로, `session_id`, unsafe env 값을 새 redaction 경로에서 다시 구현하지 않고 기존 기준과 일치시키기 위함이다.
- 기본 `npm test`에는 artifact 생성을 강제하지 않는다.
  real provider smoke는 계속 opt-in 전용이며 CI-ready offline 회귀 신호와 섞지 않는다.

## 재발 방지 포인트

- real provider smoke artifact에는 secret, token, raw debug dump, private environment value를 저장하지 않는다.
  새 필드를 추가할 때도 저장 직전 `sanitizeJson()` 적용과 테스트를 유지한다.
- `npm test` 또는 CI-ready provider offline 명령에 real smoke artifact 생성을 강제하지 않는다.
  artifact는 `npm run test:smoke:codex*` 계열 opt-in 실행의 evidence다.
- failure taxonomy 값을 바꾸면 `scripts/smoke-artifact.js`, `docs/contracts/smoke-artifact.md`, `docs/smoke-testing.md`, KG workflow를 함께 갱신한다.
- `schema_version` major 변경은 기존 summary artifact와 비교가 깨지는 변경으로 취급하고 문서와 migration 판단을 남긴다.
- timestamp 기반 `<id>`는 같은 초 동시 실행 시 충돌 가능성이 있으므로 병렬 real smoke 실행을 도입하면 ID 생성 정책을 먼저 보강한다.
