## 목표

비대화형 dogfooding, CI, agent 실행에서 Run 비용 guard가 stdin closed 상태를 기본값 `N`으로 처리해 pipeline을 중단할 때, 중단 원인과 의도적인 override 방법을 명확히 남긴다.
안전성을 해치지 않도록 기본 자동 승인은 도입하지 않는다.

## 구현 내용

- `scripts/run.js`에 `--allow-cost-overrun` CLI 플래그를 추가했다.
- 누적 비용이 임계값을 초과하고 stdin이 닫힌 경우 실행을 중단하되, 중단 원인, `--allow-cost-overrun` 재실행 예시, `max_cost_usd`와 `default_max_cost_usd` 조정 방법을 출력하게 했다.
- `--background` child process에도 `--allow-cost-overrun`이 전달되도록 인자 전파를 정렬했다.
- `test/run.test.js`에 closed stdin 중단 안내, 명시 opt-in 실행, 기존 거부 경로 안내를 검증하는 회귀 테스트를 추가했다.
- `skills/run/SKILL.md`, `skills/cost-warn/SKILL.md`, `docs/contracts/file-contracts.md`, `README.md`에 noninteractive cost guard 동작과 run-request 비용 임계값 계약을 문서화했다.

## 결정 내용

- 비용 guard 초과 자동화는 기본 자동 승인이 아니라 `--allow-cost-overrun` 명시 opt-in으로만 계속 실행한다.
- `run-request.json`의 `max_cost_usd`와 `.built/config.json`의 `default_max_cost_usd`는 임계값 조정 수단으로 유지한다.
- `--allow-cost-overrun`은 run-request 필드가 아니라 실행 시점 CLI 승인 플래그로 둔다.

## 결정 이유

- 비용 guard는 사용자가 의도하지 않은 비용 증가를 막는 안전장치이므로, noninteractive 환경이라는 이유만으로 자동 승인하면 guard의 의미가 사라진다.
- dogfooding/CI/agent 실행은 stdin prompt를 처리할 수 없으므로, 실패 출력이 원인과 다음 조치를 직접 알려야 운영자가 조용한 중단으로 오해하지 않는다.
- 임계값 조정과 일회성 override를 분리하면 반복 실행 정책은 `max_cost_usd`/`default_max_cost_usd`에 남기고, 단발성 초과 승인은 CLI 플래그로 감사 가능하게 남길 수 있다.

## 재발 방지 포인트

- 비대화형 실행 편의를 위해 비용 guard를 기본 자동 승인으로 바꾸지 않는다.
- `--allow-cost-overrun`은 비용 guard만 우회하며 provider 인증, 권한, phase 실패를 성공으로 바꾸지 않는다.
- 반복 자동화의 정상 정책은 `max_cost_usd` 또는 `default_max_cost_usd`로 조정하고, CLI flag는 의도적인 단발성 초과 승인으로 사용한다.
- run-request schema를 늘릴 때는 실행 시점 승인 플래그와 durable request contract를 섞지 않는지 확인한다.
