## 목표

provider failure taxonomy와 smoke artifact가 늘어날 때 `stderr`, app-server notification, `failure.debug_detail`, 공개 문서와 KG에 민감정보 후보가 그대로 남지 않게 한다.
사용자-facing 메시지와 runtime artifact는 조치에 필요한 근거를 유지하되 token, API key, chat id, 실제 홈 경로 같은 private identifier 후보를 sanitize해야 한다.

## 구현 내용

- `src/providers/failure.js`의 `sanitizeDebugDetail()`이 GitHub token, 주요 provider API key 환경변수 값, Telegram bot token, `chat_id`, macOS/Linux 홈 경로를 redaction하도록 확장되었다.
- `src/providers/codex.js`의 `_notificationToEvents()` error notification 경로가 app-server 오류 메시지에 `sanitizeDebugDetail()`을 적용하도록 바뀌었다.
  이 경로는 `logs/<phase>.jsonl`과 `progress.json`에 연결될 수 있어 provider raw notification을 그대로 남기지 않는다.
- `test/providers-failure.test.js`에 provider error, stderr, environment-looking string, token/path fixture가 추가되었다.
- `test/providers-codex.test.js`에 error notification 메시지가 event message와 failure detail에서 sanitize되는 회귀 테스트가 추가되었다.
- `test/docs-sensitive-check.test.js`가 추가되어 `docs/`와 `kg/` 공개 파일에서 API key, token, 실제 홈 경로 후보를 점검한다.

## 결정 사항

- runtime redaction은 `scripts/sanitize.js` 같은 문서/출력 정리 경로와 별도로 `src/providers/failure.js`의 `sanitizeDebugDetail()`에 유지한다.
  provider raw error는 사용자 메시지, `logs/<phase>.jsonl`, smoke artifact로 흘러갈 수 있으므로 provider failure 계층에서 독립 방어선을 둬야 한다.
- `failure.debug_detail`은 완전히 제거하지 않고 sanitize 후 보관한다.
  운영자는 실패 원인, 키 이름, 대략적 경로 맥락이 필요하지만 secret 값과 실제 사용자 홈 경로는 필요하지 않다.
- 홈 경로는 `~/`로 축약해 디버그 방향은 유지하고 개인 식별자만 제거한다.
- 공개 문서/KG 점검은 완전한 secret scanner가 아니라 known-risk fixture 기반 회귀 테스트로 둔다.
  실제 사용자 이름과 예시 이름을 완벽하게 구분하는 제품화 스캐너는 이 이슈의 비범위다.

## 재발 방지 포인트

- provider error event, app-server notification, stderr를 사용자-facing message나 `progress.json`/`state.json`에 연결하기 전에 `sanitizeDebugDetail()`을 적용한다.
- `failure.debug_detail`은 운영 디버그 계층이며, secret 값이나 실제 홈 경로를 보존하는 장소가 아니다.
- smoke artifact 저장 경로를 추가할 때는 artifact fixture에 token, API key, chat id, 홈 경로 후보를 포함해 redaction 회귀 테스트를 먼저 확장한다.
- `docs/`와 `kg/`에는 raw execution dump, private environment value, 실제 사용자 홈 경로를 기록하지 않는다.
- 공개 파일 scanner의 allowlist는 예시 이름과 실제 identifier 후보를 구분하기 위한 운영 장치다.
