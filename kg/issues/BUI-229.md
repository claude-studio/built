## 목표

이슈별 branch/PR/KG/merge commit mapping manifest를 정의하고, Builder PR 생성 시 기록하며
Reviewer/Recorder/Finisher가 같은 mapping을 참조해 중복 PR과 누락 KG 기록을 줄인다.

## 구현 내용

- `docs/contracts/issue-pr-mapping.md`를 추가해 이슈별 branch/PR/merge commit/KG 파일
  mapping 계약을 정의했다.
- `kg/issues/BUI-<N>.md` frontmatter를 mapping SSOT로 삼고, `branch`, `pr`,
  `merge_commit`, `kg_files` 필드의 담당 역할과 업데이트 시점을 정했다.
- Builder는 PR 생성 전 같은 이슈의 open PR을 확인하고, PR 생성 후 `branch`와 `pr`을
  기록하도록 절차가 갱신됐다.
- Reviewer는 handoff 시 mapping과 실제 PR 정보를 대조하고, Recorder는 KG 기록 후
  `kg_files`와 issue KG 본문을 갱신하며, Finisher는 merge 후 `merge_commit`을 남기도록
  역할별 KG 문서를 갱신했다.
- ADR-14로 `kg/issues/` frontmatter를 SSOT로 선택한 이유와 별도 manifest/DB schema
  변경을 선택하지 않은 이유를 남겼다.

## 결정 사항

ADR-14 참조: `kg/issues/` frontmatter를 mapping SSOT로 지정.

## 발생한 이슈

- Builder는 PR #87, branch `agent/builder/4c2e160d`, head commit `2d0f030`으로
  문서/KG 변경을 제출했다.
- Reviewer는 2026-04-26 13:40 KST 기준 PASS로 판정했다.
  변경은 문서/KG 전용이며 기존 파일과 호환된다고 보았다.
- Reviewer가 지적한 경미한 불일치는 Builder 스켈레톤의 `kg_files`가 계약서 4절보다
  먼저 채워져 있었다는 점이다.
  기능 영향은 없으며 Recorder가 최종 KG 파일 목록을 확인해 갱신했다.
- 코드 실행 테스트는 해당 없음이다.
  검증은 PR URL, branch, KG mapping 필드, 계약 문서의 역할별 절차 정합성 확인으로
  수행됐다.

## 재발 방지 포인트

- Builder는 PR 생성 전에 같은 이슈 번호 또는 같은 head branch의 open PR을 먼저 확인한다.
  mapping에 canonical PR이 있으면 새 PR을 만들지 않고 기존 branch에 추가 commit한다.
- Reviewer는 PASS 전 `kg/issues/BUI-<N>.md`의 `branch`와 `pr`이 실제 PR 정보와 맞는지
  확인한다.
- Recorder는 개발 플로우 이슈에서 main에 직접 KG를 쓰지 않고 canonical PR head branch에
  KG commit을 추가한다.
- Finisher는 merge 뒤 `merge_commit`을 기록해 PR URL, branch, merge commit, KG 파일의
  연결을 완결한다.
- mapping에는 private token, secret, 내부 host path, workspace/runtime UUID, raw execution
  log를 남기지 않는다.
