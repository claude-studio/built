## 목표

Codex provider의 실제 smoke 테스트가 기본 `npm test`나 CI에 섞이지 않도록 분리한다.
외부 인증, 비용, 네트워크, 로컬 app-server 상태에 의존하는 real provider 검증은 명시적 opt-in으로만 실행하고, 실패 시 인증, app-server, sandbox, timeout 등 조치 축을 사용자 문서에서 추적 가능하게 만든다.

## 구현 내용

- `package.json`에 Codex real smoke 전용 npm script를 추가했다.
  - `npm run test:smoke:codex:plan`: `BUILT_CODEX_PLAN_SYNTHESIS_SMOKE=1 node scripts/smoke-codex-plan-synthesis.js`
  - `npm run test:smoke:codex:do`: `BUILT_CODEX_DO_SMOKE=1 node scripts/smoke-codex-do.js`
  - `npm run test:smoke:codex`: plan smoke와 do smoke를 순서대로 실행
- `scripts/smoke-codex-plan-synthesis.js`와 `scripts/smoke-codex-do.js`에 `checkLogin()` 사전 점검을 추가했다.
  환경 변수가 없으면 skip 후 성공 종료하고, 환경 변수가 있으면 provider availability와 인증/app-server 상태를 먼저 확인한다.
- do phase smoke는 `sandbox=workspace-write` 전제를 문서화하고, `do-result.md` 미생성 또는 frontmatter 구조 불일치 시 `model_response` 축으로 실패를 분리한다.
- `docs/smoke-testing.md`를 추가해 기본 테스트, real provider smoke 실행법, 전제 조건, 실패 원인 축을 한글로 정리했다.
- `test/check.test.js`는 BUI-132에서 한글화된 실패 메시지와 맞도록 timeout/JSON parse assertion을 갱신했다.

## 결정 내용

- 기본 `npm test`는 fake/offline 테스트만 실행한다.
  real Claude/Codex provider smoke는 기본 테스트와 CI basic job에 포함하지 않는다.
- real Codex smoke는 명시적 환경 변수 또는 전용 npm script로만 실행한다.
  npm script는 환경 변수를 함께 설정하므로 문서화된 opt-in entrypoint 역할을 한다.
- smoke 실패 메시지는 문서화된 `원인축:` 키워드로 분류한다.
  이번 이슈에서 확정한 축은 `provider_unavailable`, `app-server`, `인증(auth)`, `sandbox`, `timeout`, `model_response`다.
- smoke 스크립트는 실제 provider 실행 전에 `checkLogin()` 사전 점검을 수행해 CLI 미설치, app-server 미지원, 인증 실패를 빠르게 분리한다.

## 결정 이유

- real provider smoke는 로컬 CLI 설치, 로그인 상태, 네트워크, app-server 지원 여부에 좌우되므로 기본 회귀 테스트에 넣으면 CI 안정성이 떨어진다.
- BUI-118과 BUI-119에서 시작한 opt-in smoke 패턴을 package script와 사용자 문서까지 연결해야 운영자가 같은 방식으로 재실행할 수 있다.
- 사전 점검 없이 provider 실행으로 바로 들어가면 인증 실패와 app-server 미지원, model output 실패가 모두 실행 실패로 보일 수 있다.
- 실패 축을 사용자 문서의 `원인축:` 키워드와 맞추면 issue, review, smoke log에서 같은 분류어로 원인을 추적할 수 있다.

## 재발 방지 포인트

- `npm test` 또는 기본 CI job에 real provider smoke를 추가하지 않는다.
  provider smoke는 opt-in 환경 변수나 `npm run test:smoke:*` 명령으로만 실행한다.
- smoke script를 수정할 때 환경 변수 미설정 skip 동작이 exit 0인지 먼저 확인한다.
- Codex do smoke는 파일 변경이 필요한 phase이므로 `workspace-write` sandbox 전제를 유지한다.
- 실패 메시지를 바꿀 때는 `docs/smoke-testing.md`의 실패 원인 축 표와 코드의 `원인축:` 출력이 함께 갱신되어야 한다.
- 한글 사용자 메시지 변경은 `test/check.test.js`처럼 기존 assertion과 맞는지 함께 확인한다.
