## 목표

`codex-do` preset 실행 시 Do 단계가 Codex app-server schema validation에서 시작 직후 실패하지 않게 한다.
특히 built 계약 값 `workspace-write`가 `workspaceWrite`로 변환되어 `unknown variant` 오류를 만드는 경로를 제거해야 했다.

## 구현 내용

- `src/providers/codex.js`의 `SANDBOX_TO_CODEX`를 현재 Codex app-server schema에 맞춰 kebab-case passthrough로 수정했다.
- `read-only`, `workspace-write`, `danger-full-access`를 app-server 요청에 그대로 전달하도록 변환 테이블을 정리했다.
- `runCodex` 단위 테스트에 `thread/start.params.sandbox === "workspace-write"` 요청 캡처 검증을 추가했다.
- `codex-do` preset에서 Do/Iter sandbox가 `workspace-write`로 유지되는 회귀 테스트를 추가했다.
- `kg/decisions/codex-app-server-provider-runtime-policy.md`와 `kg/workflows/codex-app-server-provider-validation.md`의 Codex sandbox 검증 기준을 최신 app-server enum에 맞춰 갱신했다.

## 결정 사항

- Codex app-server sandbox enum은 built 계약 값을 camelCase로 변환하지 않고 kebab-case로 전달한다.
  현재 app-server schema가 `read-only`, `workspace-write`, `danger-full-access`를 기대하며, camelCase 변환은 요청 schema validation 실패를 만든다.
- `danger-full-access`는 adapter enum 호환 보강으로만 기록한다.
  public provider config 계약과 parser/capability registry는 여전히 `read-only`/`workspace-write` 중심이므로 공개 계약 확장은 별도 이슈에서 다뤄야 한다.
- 실제 Codex smoke는 이번 hotfix의 merge blocker로 두지 않는다.
  환경과 인증에 의존하지만, 이번 장애 원인인 `workspaceWrite` 전달 여부는 fake app-server 요청 캡처 테스트로 직접 고정할 수 있기 때문이다.

## 발생한 이슈

- 도그푸딩에서 `todo-list-service`를 `codex-do` preset으로 실행할 때 `run-request.json`에는 `sandbox: "workspace-write"`가 정상 기록되었지만, provider adapter가 이를 `workspaceWrite`로 변환했다.
- Codex app-server는 kebab-case enum을 기대해 `workspaceWrite` unknown variant 오류를 반환했다.
- 결과적으로 `.built/features/todo-list-service/progress.json`은 `provider: "codex"`, `status: "failed"`와 해당 `last_error`를 남겼고, runtime `state.json`은 `last_failure.code: "codex_unknown"`로 실패했다.
- Builder는 PR #108, branch `agent/builder/c9195331`, head commit `803173394dbbbda93c6c827cafc8d99e56f5930b`로 hotfix를 제출했다.
- Reviewer 1차 검토는 완료 기준과 diff 범위가 맞음을 확인했지만, Codex app-server JSON-RPC protocol/runtime 연동 경로와 provider sandbox contract 변경이므로 Specialist second-review를 요청했다.
- Specialist second-review는 blocking finding이 없다고 판정했다.
  unknown sandbox fallback은 기존 direct adapter 계열 동작과 호환되며, public config 경로에서는 parser가 알 수 없는 sandbox를 먼저 거부한다고 확인했다.
- Reviewer 최종 검토는 2026-04-27 01:01 KST에 PASS했고 `SCOPED_OK Reviewer`를 남겼다.
  동일 BUI 번호의 open PR은 #108 하나이고 superseded PR은 없었다.

## 재발 방지 포인트

- Codex app-server sandbox 값은 `read-only`, `workspace-write`, `danger-full-access` kebab-case enum으로 다룬다.
  `workspaceWrite` 같은 camelCase 변환을 재도입하면 Do/Iter가 실제 작업 전에 schema validation에서 실패한다.
- `thread/start.params.sandbox === "workspace-write"` 요청 캡처 테스트를 유지한다.
  run-request 생성만 검증하면 provider adapter 변환 회귀를 놓칠 수 있다.
- `danger-full-access`를 public provider config 계약에 열려면 parser, capability registry, `docs/contracts/provider-config.md`를 함께 바꾸는 별도 결정이 필요하다.
- provider adapter는 결과 파일을 직접 쓰지 않고 표준 provider event를 runner/writer에 전달하는 책임 경계를 유지한다.
