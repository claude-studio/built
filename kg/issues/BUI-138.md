## 목표

provider 구조가 구현된 뒤 새 사용자가 문서만 보고 Claude 기본 실행과 Codex opt-in 실행을 구분할 수 있게 한다.
README와 사용자-facing 운영 문서에 provider 설정 위치, 단축형/상세형 설정, sandbox, timeout, smoke 실행, troubleshooting을 실제 parser 동작과 맞춰 정리한다.

## 구현 내용

- `README.md`에 Provider 설정 섹션을 추가해 Claude 기본값과 Codex opt-in 정책을 비교표로 설명했다.
- provider 설정 위치를 `.built/runtime/runs/<feature>/run-request.json`의 `providers` 필드로 명시했다.
  `.built/config.json`에 `providers`를 넣으면 `/built:validate`에서 `unknown key(s): 'providers'` 오류가 난다는 점도 함께 기록했다.
- `docs/ops/provider-setup-guide.md`를 새로 작성해 기본 개념, 설정 위치, 단축형/상세형 예시, phase별 시나리오, sandbox 정책, timeout, smoke 테스트, 오류 메시지 참조를 한 곳에 모았다.
- Codex `do`/`iter` phase는 `sandbox: "workspace-write"`가 필수이고, `plan_synthesis`/`check`/`report`는 read-only 사용이 가능하다는 정책을 문서화했다.
- smoke 실행은 실제 provider를 호출하는 수동 opt-in 경로로 설명하고, `npm run test:smoke:codex:plan`, `npm run test:smoke:codex:do`, `npm run test:smoke:codex` 명령을 안내했다.
- 오류 메시지 표를 `src/providers/config.js`, `scripts/validate.js`, smoke 실패 원인축과 맞춰 정리했다.

## 결정 사항

- provider 설정의 사용자-facing SSOT는 `run-request.json`의 `providers` 필드로 둔다.
  repo 전역 `.built/config.json`은 built 기본 설정 검증 대상이며 provider phase 선택 위치가 아니다.
- Codex는 기본값이 아니라 phase별 opt-in이다.
  설정하지 않은 phase는 Claude 기본값으로 실행되므로, 문서 예시는 전체 provider 전환이 아니라 필요한 phase만 지정하는 방식으로 작성한다.
- 단축형 예시는 지원하되, 파일을 쓰는 Codex `do`/`iter`에는 상세형으로 `sandbox: "workspace-write"`를 명시하는 예시를 우선한다.
- `providers.plan_synthesis`를 설정하면 `plan_synthesis` phase가 자동 활성화된다.
  `plan_synthesis: true`는 함께 쓸 수 있지만 필수 조건으로 문서화하지 않는다.
- 실제 Codex 연결 확인은 기본 테스트가 아니라 smoke 테스트로 분리한다.
  Codex CLI 설치, 로그인, app-server 지원 여부에 따라 실패할 수 있으므로 troubleshooting 표에서 원인축과 조치 방법을 안내한다.

## 재발 방지 포인트

- `providers`는 `.built/config.json`이 아니라 `run-request.json`에 둔다.
  config.json에 넣으면 `scripts/validate.js`의 known key 검증에서 `unknown key(s): 'providers'` 오류가 발생한다.
- provider 문서를 수정할 때는 `docs/contracts/provider-config.md`뿐 아니라 실제 parser와 runtime activation 조건을 함께 확인한다.
  특히 `scripts/run.js`의 `hasPlanSynthesisEnabled()`처럼 phase 활성화 로직이 parser와 별도인 경우가 있다.
- `providers.plan_synthesis` 단독 설정은 plan_synthesis phase를 활성화한다.
  `plan_synthesis: true`를 필수로 안내하면 사용자가 불필요한 중복 설정을 요구받게 된다.
- Codex `do`/`iter` 문서 예시는 `workspace-write` sandbox를 포함해야 한다.
  read-only 설정은 파일 변경이 불가능해 parser 또는 runner 단계에서 즉시 실패해야 한다.
- 오류 메시지를 문서에 적을 때는 전체 문자열이 아니라 앞부분만 인용하는 경우 부분 인용임을 독자가 오해하지 않게 원인과 조치 중심으로 적는다.
