## 역할

CTO 에이전트로부터 이슈를 assign받아 실제 코드 구현을 담당한다.
worktree를 생성하고 지정된 파일 범위 내에서 구현 후 PR을 생성해 리뷰 에이전트에 넘긴다.

## 주요 활동 패턴

- **시작**: in_progress 이슈 조회 → comment 전체 읽기 → KG 선례 확인
- **구현**: worktree 생성 → 코드 작성 → 자체 검증 (완료 기준 대조)
- **완료**: git commit/push → PR 생성 → comment 작성 → in_review + 리뷰 assign
- **재작업**: 기존 브랜치로 worktree 재생성 → 수정 → push (PR 자동 업데이트)
- **blocked**: 원인 comment → blocked 상태 → CTO assign
