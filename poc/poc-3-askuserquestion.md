- **이슈**: BUI-4
- **브랜치**: poc-3
- **검증일**: 2026-04-24
- **담당**: 개발 에이전트

---

## 목표

Plan 인터뷰 플로우에서 AskUserQuestion을 여러 번 연속 호출해 사용자 응답을 단계적으로 받을 수 있는지 검증한다.

---

## 검증 결과

### 1. AskUserQuestion 연속 호출 (최소 3회) 동작 확인

**결과: 성공**

같은 세션 내에서 AskUserQuestion을 3회 연속 호출했다. 각 호출은 독립적으로 동작했으며, 이전 호출의 완료 여부와 무관하게 다음 호출이 정상 실행됐다.

- 1회차: 호출 성공, 사용자 응답 대기 → 결과 수신
- 2회차: 1회차 결과 수신 후 즉시 호출 성공
- 3회차: 2회차 결과 수신 후 즉시 호출 성공

**핵심 관찰**: AskUserQuestion은 동기적(blocking) 도구다. 한 번 호출하면 사용자가 응답하거나 취소할 때까지 다음 코드로 진행하지 않는다. 이는 인터뷰 플로우에서 Q1 → 응답 → Q2 → 응답 → Q3 순서를 자연스럽게 구현할 수 있다는 의미다.

### 2. 이전 응답을 다음 질문에 활용 가능한지 확인 (컨텍스트 유지 여부)

**결과: 가능**

AskUserQuestion 도구의 결과값은 Claude 메시지 히스토리에 tool result로 기록된다. 이후 호출 시점에 에이전트는 이전 모든 tool result를 컨텍스트로 보유하므로, 이전 응답을 다음 질문에 반영하는 것이 가능하다.

예시 패턴:
```
Q1: "어떤 기능을 만들고 싶으신가요?" → 사용자: "로그인 시스템"
Q2: "로그인 시스템의 주요 사용자는 누구인가요?" (Q1 응답 "로그인 시스템"을 질문에 반영)
Q3: "로그인 실패 시 처리 방식은?" (Q1, Q2 응답 모두 컨텍스트에 보유)
```

### 3. 호출 간 상태 유지 방식

**방식: Claude 메시지 히스토리 (자동)**

별도의 상태 저장 메커니즘이 필요 없다. 모든 AskUserQuestion 호출과 그 결과는 Claude 대화 메시지 히스토리에 순서대로 누적된다. 에이전트가 n번째 질문을 할 때는 1~(n-1)번째 질문과 답변이 모두 컨텍스트에 존재한다.

**제약사항**:
- 사용자가 응답하지 않고 취소하면 tool result가 error 상태로 기록되지만, 에이전트는 계속 실행 가능
- 한 번에 1~4개의 질문을 하나의 호출에 묶을 수 있다 (multiSelect 지원)
- 세션이 끊기면 상태가 사라진다 (중간 저장 필요 시 plan-draft.md 활용)

### 4. /built:plan 6단계 인터뷰 구현 가능 여부 결론

**결론: 구현 가능**

BUILT-DESIGN.md §6의 6단계 플로우는 AskUserQuestion 다중 연속 호출로 완전히 구현 가능하다.

| 단계 | 호출 방식 |
|------|-----------|
| Phase 1: Intent | AskUserQuestion 4~6회 (Purpose, Scope, Content, Constraints 각각 또는 묶어서) |
| Phase 2: Architecture | AskUserQuestion 1~2회 (접근법 선택, 근거) |
| Phase 3: Build Plan | 자동 생성 후 AskUserQuestion 1회 (확인/수정) |
| Phase 4: Spec Review | AskUserQuestion 1회 (전체 확인 또는 섹션별 재작업 요청) |
| Phase 5: Save | 자동 저장 |

Phase별로 묶어서 호출하는 것이 UX 면에서 유리하다. 예를 들어 Phase 1에서 4개 항목을 개별 호출하면 4번의 턴이 발생하지만, 관련 항목을 묶어 한 번 호출하면 1~2번의 턴으로 처리 가능하다.

## 구현 방향 권고

### 권고 패턴: 단계별 묶음 + 개별 심화

```
Phase 1 (1회 호출, multiSelect 없이 핵심 질문 1개):
  "이 기능의 목적과 성공 기준을 설명해주세요."
  → 응답 받고 Claude가 파싱

Phase 1 (후속, 필요시):
  "다음 중 이 기능의 Anti-goal에 해당하는 것을 선택해주세요." (multiSelect: true)
  → 여러 항목 선택 가능

Phase 2 (1회 호출):
  "다음 3가지 접근법 중 선호하는 방향은?" (options 3개)

Phase 4 (1회 호출):
  "Spec을 확인해주세요. 수정이 필요한 단계는?" (multiSelect: true)
```

### 주의사항

1. **세션 지속성**: AskUserQuestion은 세션 내에서만 상태를 유지한다. 사용자가 Plan 도중 세션을 끊으면 처음부터 다시 시작해야 한다. `plan-draft.md`에 중간 결과를 저장하여 재개 가능하도록 설계해야 한다.

2. **응답 취소 처리**: 사용자가 질문을 취소하면 에러가 반환되지만 세션은 계속된다. SKILL.md에서 취소 시 처리 로직을 명시해야 한다.

3. **질문 수 제한**: 한 번의 호출에서 최대 4개 질문까지 가능하다. Phase 1처럼 질문이 많은 단계는 여러 번 나눠서 호출하거나, 선택지 방식으로 구조화해야 한다.

4. **"standard/normal" 답변 방지**: 선택지 설계 시 구체적인 옵션을 제공하고, Other(사용자 직접 입력)를 통해 커스텀 응답을 유도해야 한다.
